SOTA

Data analytics platform for machine learning scientists

Project Introduction

After learning a lot from existing data and building their own models, data scientists need data visualization tools with great user experience to help them visualize the performance of their existing models. What's more, work together with other team members to share experimental results and continuously update and revise each other's models as a way to build the best machine learning models.

Company and Time

SEA | Sep 2021

Role

User researcher & UX designer

Result

We built a machine learning data analysis platform for SEA AI Labs and NUS to protect the privacy and security of the company's internal data, enhance teamwork, visualize machine learning model results, and continuously optimize the company's internal algorithms for games, e-commerce, and other products to provide a more intelligent user experience.

User research

Existing user behavior analysis

How do users work currently? How is the machine learning model built? What else needs to be done besides visualization?

Understand what the user is currently working on, to help think about what the machine learning data analytics platform needs to achieve from a global perspective. Visualization is only part of the total process, it is more important to understand how the user's entire workflow works before visualization.

Conduct qualitative user interviews

Digging deeper into user requirements, Understanding requirements prioritization, Establishing MVP

It is not enough just to understand the current workflow of users, through qualitative research, organize user interviews, more in-depth understanding of the reasons behind user needs, the actual goals to be achieved by users, the user's internal ranking of priorities, the establishment of MVP and iterative planning

  • Step 1: Develop a user research plan
  • Step 2: Writing user interview scripts
  • Step 3: Conduct user interview session
  • Step 4: Document and analyze the results of user interviews
  • Step 5: Output interview report. In-depth discussion with PM on product direction

Persona

Create user personas to help teams use empathy to understand users

Based on the results of previous user interviews, Barry Wilson was created as a user persona to help the team build a unified perception of the target users.

User journey map

Summarize the workflow, analyze user pain points, and outline the design direction

Through user journey mapping, help self-summarize all user pain points encountered by users throughout the journey, and determine the basic requirement priorities through emotional changes, and communicate with PM, development through visualization to establish a common team recognition of user requirements.

Function Modules

Summarize the product modules. Determine the basic layout

Based on the results of previous user interviews and user analysis, and after multiple discussions with PMs, we finally confirmed the functional modules of MVP after striking the best balance between product, technology and users.

Design solutions

Build the basic information architecture

Thinking, mapping, confirming product maps

Before starting into the real interaction draft drawing, draw a good product infrastructure diagram to help myself organize my thoughts and determine the basic direction of the product.

Interaction design

Collaborate with PMs to produce detailed interaction drafts

After confirming the requirements and functional modules, we started the first phase of interaction design work, during which we worked closely with PM and other team members to improve the interaction details, interaction logic and design rationality.

User interview during interaction design phase

Conduct the first round of user validation as well as interviews with users before moving on to visual design to reduce labor costs, ensure that the product as a whole meets user expectations, and improve product usability.

Visual design

Collaborate with UI designers to output the final visual design.

Product Iteration

Keep close communication with users

We strive to maintain close contact with users through various channels to collect their requirements.

  • Create user groups to keep in touch with users instantly.
  • Create a public list of user requirements so that users can fill in their problems and requirements at any time
  • Expose the product road map, allowing users to participate in the overall life cycle of product iterations.

Iteration plan

Develop iterative plans that are detailed in the near term and relatively broad in the long term based on resources, needs, and values.

Create a user story map, let developers, PMs, and QAs participate in the map, so that the team has a common vision, and unify the team's common perception of the near-term and long-term goals.

  • Resources, user needs, business value analysis.
  • Organize user story map workshops.
  • PM, designers, developers, and users repeatedly communicate if changes are needed.
  • Determine the final release as well as the iteration plan.
