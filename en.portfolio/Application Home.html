Application Home

Unified homepage for all Autodesk prodcuts

Project Introduction

The home page is the first page that users see when they launch an Autodesk product. Ideally, all Autodesk products should provide a consistent experience for users. Currently, each Autodesk product has a very different homepage experience. In addition, the homepage provides functionality that does not match the tasks the user needs to perform, contains ineffective information, and does not have a reasonable distribution of valid information on the page that meets the user's expectations.
The Application Home program brings together all the features and content that users need before they can begin their daily work, highlighting the most common needs of users and providing flexibility and extensibility for the different needs of each product, creating a sense of space for users and increasing the impact of Autodesk brand consistency at the same time.

Company and Time

Autodesk | Mar 2018

Role

User researcher, interaction designer and visual designer, and responsible for post-production of client-facing videos

Result

Design product homepages for Revit and provide extensible product frameworks for all Autodesk products. All products can build their own product homepages with Autodesk brand consistency in a very short time.

Revit Home requirement analysis

Business goals analysis

Understand the business goals the company is trying to achieve

  • Reconstruct the information architecture, interaction and visual style of Revit home page, so that the home page is more meet user targets in the home page, and remove the redundant information.
  • Provide Revit users with a unified cloud model browsing, access and management experience on the home page to easily access and share design data in the cloud with all project collaborators. And guide users to transition from local to cloud models.
  • Provide an extensible home page framework to allow more products to build home pages using a unified framework.

Existing user behavior analysis

Mapping out today's workflow and systematically understanding the problems users are currently experiencing and where our opportunities lie.

Summary of user problems and solutions

User problems

  • Currently in Revit if you want to access the cloud project, editing the cloud model is a very painful experience, users need to go to multiple different open models, edit the model, restore the version and share the model.
  • Revit Home and BIM 360 look like products from two different companies, with completely different visuals, interaction methods, information architecture, etc.
  • The home page design lacks rationality and needs a more modern visual style, a more interactive approach in line with user habits, and an information architecture more in line with user mental models.

Solutions

  • By providing customers with a unified and integrated design collaboration and management solution and model viewing and management experience, they can easily access and share design files whether they are working internally or externally with others.
  • Provide users with access to more Autodesk cloud services throughout the AEC lifecycle, such as cloud rendering, BIM 360 docs cloud-based document management.
  • Reconstruct the information architecture, interaction and visual style of the homepage to make it more in line with users' needs and expectations.

From Revit Home to Autodesk Application Home

Stakeholder map

Since this is a company-level, cross-product project, we needed to identify which products would be ready to use the Application Home framework in Phase 1 and develop a stakeholder map to ensure smooth communication and interoperability. The teams that will be included in the Phase 1 project include Revit, Civil 3D, BIM 360, AutoCAD and the HIG team.

Company-level product architecture

Consider the architecture of the whole product from the perspective of flexibility, shareability, extensibility, and adoptability, and how to build a complete ecosystem of Application Home, imagine that the whole Application Home is a pool from which everyone can freely take the parts they need and use them for themselves, or point out the shortcomings and make them complete.

Information architecture

When designing the information architecture, we need to understand the priority of the user's tasks to be achieved on the home page, the user's operating habits, and the user's mental model by means of user research, so as to design the overall information architecture.

  • Resume work - Users returning to the product want to continue working where they left off.
  • Start a new work - Users want to start a new work, start with a completely blank file, start with a template, or start with their own previous reusable projects. These documents are available from local settings, cloud storage, intranets, etc.
  • Learning - Users want to explore the product, improve their skills, discover new changes that occur in the product or receive notifications about them.
  • Engagement - Users want to participate in the customer community, provide feedback or contact customers for help or support.
