Coral Design System

Design system for all designers in Sea/Labs

Project Introduction

As more and more designers join the team, more and more projects need to support, and more and more roles need to collaborate. How to improve the team efficiency, keep consistency and ensure high quality become more and more critical. Coral design system initiative by UX team then expanded and involved UI team, developer team, and PM team to work on that together. Not only do UX designers benefit from it, but also UI design, product team, and developer team reduce the efforts, improve efficiency, and make everyone can speak and stand on the same page.

Company and Time

Sea | 2021-2022

Role

Component contributor, Component reviewer

Result

Complete the internal component library for the design team in Sea, improve the team efficiency, and speed up the UX design, UI design, and development process.

Process of Coral

Iteration phase

At the very beginning of Coral, we made an iteration plan for the entire system. To make the short -term plan as detail as possible, prepare to change the long-term plan any time. We modify the plan all the way according to requirement change.

Components working process

Catch up and review meeting

Schedule bi-weekly catch up and review meeting to:

  • Catch up the latest progress of ongoing components.
  • UX review about the completed UX of component.
  • Problem discussion if we met any problems during the UX design.
  • Latest updates and feedback from designer, developers, PMs, and leadership team.

Overview checklist

Since tons of components work in progress or in the plan list, so a checklist for us to have an overview of the big picture is very important.

Issue tracker

During the process, we well prepared for any issue, sometimes we can solve it immediately, sometimes we have to solve it later. In that case, we need a place to track all issue for different component and define the priority of it and when to solve it.

Conduct sharing session

Conduct sharing sessions to share the components in our library to allow user know:

  • How to use the component?
  • The difference between similar component.
  • Gather feedback and suggestion.

Add more components

According to the projects requirements, by using the component submission process, coral team will add more components, there are the components in the short-term plan:

  • Data display
  • Comment
  • Toolbar
  • Login
  • Card

Provide support

As the PIC of Coral system and Coral component, we have to provide support for different aspect:

  • Solve problems if user don't know how to select correct component.
  • Modify the component if we find new problems.
  • Spreading and allow more users know the Coral and how to use it.
